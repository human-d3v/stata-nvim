## Syntax

`exlogistic depvar`
[indepvars](http://www.stata.com/help.cgi?indepvars)
_\[`if`\] \[`in`\]_ \[`weight`\] \[`,`
`options`\]

`depvar` can be specified as a zero or nonzero variable or the number of
positive outcomes within each trial. For a zero or nonzero variable,
zero indicates failure and nonzero indicates success. To specify
`depvar` as the number of positive outcomes, you must also specify
`binomial(varname,#)`.

| See [<strong>[R]</strong> exlogistic postestimation](http://www.stata.com/help.cgi?exlogistic_postestimation) for features available after estimation. <span options="menu">{marker menu}_{nobreak None} {title None:Menu} {phang None} **Statistics &gt; Exact statistics &gt; Exact logistic regression** <span options="description">{marker description}_{nobreak None} {title None:Description} {pstd None} `exlogistic` fits an exact logistic regression model, which produces more accurate inference in small samples than the standard maximum-likelihood-based logistic regression estimator. It can also better deal with completely determined outcomes. `exlogistic` with the `group(varname)` option conditions on the number of positive outcomes within stratum and is an alternative to the conditional (fixed-effects) logistic regression estimator. {pstd None} Unlike Stata's other estimation commands, `exlogistic` must perform hypothesis tests during estimation rather than during postestimation with standard postestimation commands. <span options="linkspdf">{marker linkspdf}_{nobreak None} {title None:Links to PDF documentation} [Quick start](http://www.stata.com/manuals14/rexlogisticquickstart.pdf) [Remarks and examples](http://www.stata.com/manuals14/rexlogisticremarksandexamples.pdf) [Methods and formulas](http://www.stata.com/manuals14/rexlogisticmethodsandformulas.pdf) {pstd None} The above sections are not included in this help file. <span options="options">{marker options}_{nobreak None} {title None:Options} {dlgtab None:Model} <span options="condvars">{marker condvars}_{nobreak None} {phang None} `condvars(varlist)` specifies variables whose parameter estimates are not of interest to you. You can save substantial computer time and memory moving such variables from [indepvars](http://www.stata.com/help.cgi?indepvars) to `condvars()`. Understand that you will get the same results for `x1` and `x3` whether you type `. exlogistic y x1 x2 x3 x4` {pmore None} or `. exlogistic y x1 x3, condvars(x2 x4)` {phang None} `group(varname)` specifies the variable defining the strata, if any. A constant term is assumed for each stratum identified in `varname`, and the sufficient statistics for [indepvars](http://www.stata.com/help.cgi?indepvars) are conditioned on the observed number of successes within each group. This makes the model estimated equivalent to that estimated by `clogit`, Stata's conditional logistic regression command (see [<strong>[R]</strong> clogit](http://www.stata.com/help.cgi?clogit)). `group()` may not be specified with `noconstant` or `estconstant`. <span options="binomial()">{marker binomial()}_{nobreak None} {phang None} `binomial(`[varname](http://www.stata.com/help.cgi?varname)\|`#)` indicates that the data are in binomial form and [depvar](http://www.stata.com/help.cgi?depvar) contains the number of successes. `varname` contains the number of trials for each observation. If all observations have the same number of trials, you can instead specify the number as an integer. The number of trials must be a positive integer at least as great as the number of successes. If `binomial()` is not specified, the data are assumed to be Bernoulli, meaning that `depvar` equaling zero or nonzero records one failure or success. {phang None} `estconstant` estimates the constant term. By default, the models are assumed to have an intercept (constant), but the value of the intercept is not calculated. That is, the conditional distribution of the sufficient statistics for the [indepvars](http://www.stata.com/help.cgi?indepvars) is computed given the number of successes in [depvar](http://www.stata.com/help.cgi?depvar), thus conditioning out the constant term of the model. Use `estconstant` if you want the estimate of the intercept reported. `estconstant` may not be specified with `group()`. {phang None} `noconstant`; see [<strong>[R] estimation options</strong>](estimation%20options##noconstant). `noconstant` may not be specified with `group()`. {dlgtab None:Terms} {phang None} <span options="terms">{marker terms}_{nobreak None} `terms(termname = variable` ... `variable`\[`, termname = variable` ... `variable` ...\]`)` defines additional terms of the model on which you want `exlogistic` to perform joint-significance hypothesis tests. By default, `exlogistic` reports tests individually on each variable in [indepvars](http://www.stata.com/help.cgi?indepvars). For instance, if variables `x1` and `x3` are in `indepvars`, and you want to jointly test their significance, specify `terms(t1=x1 x3)`. To also test the joint significance of `x2` and `x4`, specify `terms(t1=x1 x3, t2=x2 x4)`. Each variable can be assigned to only one term. {pmore None} Joint tests are computed only for the conditional scores tests and the conditional probabilities tests. See [<strong>test()</strong>](#test) below. {dlgtab None:Options} {phang None} <span options="memory">{marker memory}_ `memory(#`\[`b`\|`k`\|`m`\|`g`\]`)` sets a limit on the amount of memory `exlogistic` can use when computing the conditional distribution of the parameter sufficient statistics. The default is `memory(10m)`, where `m` stands for megabyte, or 1,048,576 bytes. The following are also available: `b` stands for byte; `k` stands for kilobyte, which is equal to 1,024 bytes; and `g` stands for gigabyte, which is equal to 1,024 megabytes. The minimum setting allowed is `1m` and the maximum is `2048m` or `2g`, but do not attempt to use more memory than is available on your computer. Also see the [<strong>technical note</strong>](#enumerations) on counting the conditional distribution. {phang None} <span options="saving">{marker saving}_ `saving(filename`\[`, replace`\]`)` saves the joint conditional distribution to `filename`. This distribution is conditioned on those variables specified in `condvars()`. Use `replace` to replace an existing file with `filename`. A Stata data file is created containing all the feasible values of the parameter sufficient statistics. The variable names are the same as those in [indepvars](http://www.stata.com/help.cgi?indepvars), in addition to a variable named `_f_` containing the feasible value frequencies (sometimes referred to as the condition numbers). {dlgtab None:Reporting} {phang None} `level(#)`; see [<strong>[R] estimation options</strong>](estimation%20options##level()). The `level(#)` option will not work on replay because confidence intervals are based on estimator-specific enumerations. To change the confidence level, you must refit the model. {phang None} `coef` reports the estimated coefficients rather than odds ratios (exponentiated coefficients). `coef` may be specified when the model is fit or upon replay. `coef` affects only how results are displayed and not how they are estimated. {phang None} <span options="test">{marker test}_ `test(sufficient`\|`score`\|`probability)` reports the p-value associated with the observed sufficient statistics, the conditional scores tests, or the conditional probabilities tests, respectively. The default is `test(sufficient)`. If `terms()` is included in the specification, the conditional scores test and the conditional probabilities test are applied to each term providing conditional inference for several parameters simultaneously. All the statistics are computed at estimation time regardless of which is specified. Each statistic may thus also be displayed postestimation without having to refit the model; see [<strong>[R] exlogistic postestimation</strong>](http://www.stata.com/help.cgi?exlogistic%20postestimation). {phang None} `mue(varlist)` specifies that median unbiased estimates (MUEs) be reported for the variables in `varlist`. By default, the conditional maximum likelihood estimates (CMLEs) are reported, except for those parameters for which the CMLEs are infinite. Specify `mue(_all)` if you want MUEs for all the [indepvars](http://www.stata.com/help.cgi?indepvars). {phang None} `midp` instructs `exlogistic` to use the mid-p-value rule when computing the MUEs, p-values, and confidence intervals. This adjustment is for the discreteness of the distribution and halves the value of the discrete probability of the observed statistic before adding it to the p-value. The mid-p-value rule cannot be applied to MUEs whose corresponding parameter CMLE is infinite. {phang None} `nolog` prevents the display of the enumeration log. By default, the enumeration log is displayed, showing the progress of computing the conditional distribution of the sufficient statistics. <span options="technote">{marker technote}_{nobreak None} <span options="enumerations">{marker enumerations}_{nobreak None} {title None:Technical note} {pstd None} The `memory(#)` option limits the amount of memory that `exlogistic` will consume when computing the conditional distribution of the parameter sufficient statistics. `memory()` is independent of the data maximum memory setting (see `set max_memory` in [<strong>[D] memory</strong>](http://www.stata.com/help.cgi?memory)), and it is possible for `exlogistic` to exceed the memory limit specified in `set max_memory` without terminating. By default, a log is provided that displays the number of enumerations (the size of the conditional distribution) after processing each observation. Typically, you will see the number of enumerations increase, and then at some point they will decrease as the multivariate shift algorithm ([<strong>Hirji, Mehta, and Patel 1987</strong>](#HMP1987)) determines that some of the enumerations cannot achieve the observed sufficient statistics of the conditioning variables. When the algorithm is complete, however, it is necessary to store the conditional distribution of the parameter sufficient statistics as a dataset.
