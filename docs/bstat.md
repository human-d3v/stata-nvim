| See [<strong>[R]</strong> bootstrap postestimation](http://www.stata.com/help.cgi?bootstrap_postestimation) for features available after estimation. <span options="menu">{marker menu}_{nobreak None} {title None:Menu} {phang None} **Statistics &gt; Resampling &gt; Report bootstrap results** <span options="description">{marker description}_{nobreak None} {title None:Description} {pstd None} `bstat` is a programmer's command that computes and displays estimation results from bootstrap statistics. For each variable in [varlist](http://www.stata.com/help.cgi?varlist), `bstat` computes a covariance matrix, estimates bias, and constructs normal confidence intervals (CIs), percentile CIs, bias-corrected (BC) CIs, and bias-corrected and accelerated (BCa) CIs using a bootstrap dataset in memory or on disk. The computed CIs can be displayed using `estat bootstrap`; see [<strong>[R]</strong> bootstrap postestimation](http://www.stata.com/help.cgi?bootstrap_postestimation). {pstd None} `bstat` without `varlist` replays results from the last bootstrap estimation when results are stored in `e()`. <span options="linkspdf">{marker linkspdf}_{nobreak None} {title None:Links to PDF documentation} [Remarks and examples](http://www.stata.com/manuals14/rbstatremarksandexamples.pdf) {pstd None} The above sections are not included in this help file. <span options="options">{marker options}_{nobreak None} {title None:Options} {dlgtab None:Main} {phang None} `stat(vector)` specifies the observed value of each statistic (that is, the value of the statistic using the original dataset). {phang None} `accel(vector)` specifies the acceleration of each statistic, which is used to construct BCa CIs. {phang None} `ties` specifies that `bstat` adjust for ties in the replicate values when computing the median bias used to construct BC and BCa CIs. {phang None} `mse` specifies that `bstat` compute the variance by using deviations of the replicates from the observed value of the statistics. By default, `bstat` computes the variance by using deviations from the average of the replicates. {dlgtab None:Reporting} {phang None} `level(#)`; see [<strong>[R] estimation options</strong>](estimation%20options##level()). {phang None} `n(#)` specifies the number of observations from which bootstrap samples were taken. This value is used in no calculations but improves the table header when this information is not saved in the bootstrap dataset. {phang None} `notable` suppresses the display of the output table. {phang None} `noheader` suppresses the display of the table header. This option implies `nolegend`. {phang None} `nolegend` suppresses the display of the table legend. {phang None} `verbose` specifies that the full table legend be displayed. By default, coefficients and standard errors are not displayed. {phang None} `title(text)` specifies a title to be displayed above the table of bootstrap results; the default title is `Bootstrap results`. <span options="display_options">{marker display\_options}_{nobreak None} {phang None} `display_options`: `cformat(%fmt)`, `pformat(%fmt)`, `sformat(%fmt)`, and `nolstretch`; see [<strong>[R] estimation options</strong>](estimation%20options##display_options). <span options="example">{marker example}_{nobreak None} {title None:Example} {pstd None}Setup |                   |                                                             |

Save the acceleration statistic vector

    matrix a = e(accel)

Save the estimated coefficients from the full sample

    matrix b = e(b)

Replay the bootstrap results by using the saved full sample estimate
vector and the saved acceleration vector

    bstat using bstat, stat(b) accel(a)
