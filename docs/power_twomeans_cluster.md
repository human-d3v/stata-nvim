## Syntax

Compute numbers of clusters

`power twomeans m1 m2,` {`mspec`
| `nspec cluster`<span options=")-">{c
)-}_ \[`options`\]

Compute cluster sizes

`power twomeans m1 m2, kspec` \[`options`\]

Compute power

`power twomeans m1 m2, kspec` <span options="-(">{c
-(}_`mspec` | `nspec`<span
options=")-">{c )-}_ \[`options`\]

Compute effect size and experimental-group mean

`power twomeans m1, kspec` <span options="-(">{c
-(}_`mspec` | `nspec`<span
options=")-">{c )-}_ `power(numlist)` \[`options`\]

where `m1` is the mean in the control (reference) group and `m2` is the
mean in the experimental (comparison) group. `m1` and `m2` may each be
specified either as one number or as a list of values in parentheses
(see
[<strong>numlist</strong>](http://www.stata.com/help.cgi?numlist)).

`kspec` is one of

`k1()k2()k1()kratio()k2()kratio()`

`mspec` is one of

`m1()m2()m1()mratio()m2()mratio()`

`nspec` is one of

`n1()n2()n1()nratio()n2()nratio()`

The sample-size determination thus consists of the determination of the numbers of clusters given cluster sizes or the determination of cluster sizes given the numbers of clusters. For a general discussion of using `power twomeans`, see [<strong>[PSS]</strong> power twomeans](http://www.stata.com/help.cgi?power_twomeans). The discussion below is specific to the CRD. {pstd None} If you specify the `cluster` option, include `k1()` or `k2()` to specify the number of clusters or include `m1()` or `m2()` to specify the cluster size, the [<strong>power twomeans</strong>](http://www.stata.com/help.cgi?power%20twomeans) command will perform computations for a two-sample means test in a CRD. {pstd None} All computations are performed for a two-sided hypothesis test where, by default, the significance level is set to 0.05. You may change the significance level by specifying the `alpha()` option. You can specify the `onesided` option to request a one-sided test. By default, all computations assume a balanced or equal-allocation design, meaning equal numbers of clusters and cluster sizes in both groups; see [**\[PSS\] unbalanced designs**](http://www.stata.com/manuals14/pssunbalanceddesigns.pdf) for a description of how to specify an unbalanced design. {pstd None} To compute the number of clusters in both groups, you must provide cluster sizes for both groups. There are multiple ways to supply cluster sizes, but the most common is to specify the cluster size of the control group in the `m1()` option and the cluster size of the experimental group in the `m2()` option. See [<var class="command">mspec</var><strong></strong>](#mspec) and [<var class="command">nspec</var><strong></strong>](#nspec) under [<var class="command">Syntax</var><strong></strong>](#syntax) for other specifications. Other parameters are specified as described in [PSS powertwomeansRemarksandexamplesUsingpowertwomeans`Using power twomeans`](http://www.stata.com/manuals14/psspowertwomeansremarksandexamplesusingpowertwomeans.pdf) in **\[PSS\] power twomeans**. {pstd None} You can also compute the number of clusters or the cluster size in one of the groups given the number of clusters or the cluster size in the other group by specifying the `compute()` option.
